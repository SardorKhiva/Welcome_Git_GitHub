1. GitHub bilan tanishish

    GitHub - bu dasturchilar uchun onlayn platforma bo'lib, u kodni saqlash, boshqarish va hamkorlik qilish imkonini beradi.

  2. GitHub nima?

    GitHub - bu Git versiya nazorati tizimidan foydalanib, kodni saqlash va boshqarish uchun mo'ljallangan onlayn platforma. U dasturchilar uchun hamkorlik qilish, kodni baham ko'rish va loyihalarni boshqarish imkonini beradi.

  3. GitHub qanday ishlaydi?

    GitHub, Git versiya nazorati tizimidan foydalanib, kodni saqlash va boshqarish uchun mo'ljallangan. Dasturchilar o'z kodlarini GitHub ga yuklab, ularni boshqarish va hamkorlik qilish imkoniyatiga ega bo'lishadi.

  4. GitHub asosiy tushunchalari

    • Repository (saqlov): Kod va fayllarni saqlash uchun mo'ljallangan joy.
    • Branch (tarmoq): Loyihaning parallel versiyasi, o'zgarishlarni sinab ko'rish uchun ishlatiladi.
    • Commit: O'zgarishlarni saqlash va tarixga qo'shish jarayoni.
    • Pull so'rov: O'zgarishlarni asosiy branchga qo'shish uchun yuboriladigan so'rov.

  5. GitHub asosiy xususiyatlari

    • Versiya nazorati
    • Hamkorlik
    • Pull so'rovlar
    • Issues va project management

  6. GitHub bilan ishlash

    GitHub bilan ishlash uchun quyidagi asosiy amallarni bajarish kerak:

    1. Hisob yaratish
    2. Repository (saqlov) yaratish
    3. Kodni yuklash yoki klonlash
    4. O'zgarishlarni amalga oshirish va commit qilish
    5. Pull so'rov yuborish

  7. GitHub bilan bog'liq asosiy buyruqlar

    git clone [repository URL] - Repository ni klonlash
        
  8. git add [file] - O'zgarishlarni qo'shish
  9. git commit -m "Commit xabari" - O'zgarishlarni saqlash
  10. git push - O'zgarishlarni GitHub ga yuborish
  11. git pull - GitHub dan o'zgarishlarni olish
  12. git branch - Branchlarni ko'rish
  13. git checkout [branch] - Branchga o'tish
  14. git merge [branch] - Branchlarni birlashtirish
  15. git status - Hozirgi holatni ko'rish
  16. git log - Commit tarixini ko'rish
  17. git remote -v - Remote repositorylarni ko'rish
  18. git fetch - Remote repositorydan o'zgarishlarni olish
  19. git reset --hard [commit] - O'zgarishlarni belgilangan commitga qaytarish
